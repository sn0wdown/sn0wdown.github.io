Samba远程代码执行(CVE-2017-7494)

发表于 2017-09-14 | 分类于 漏洞测试 | 阅读次数

参考文档：
http://www.itdadao.com/articles/c15a227580p0.html
http://www.bkjia.com/xtzh/926935.html

$ tar –zxvf samba-4.5.9.tar.gz    
$ cd samba-4.5.9    
$ ./configure

执行

没有问题的话，执行./configure显示成功

然后编译安装（我们在./configure未指定路径的话，默认安装在/usr/local/samba下） 

$ make && make install

这里要等很久… ok

接下来的操作

1）复制源码包中的配置文件smb.conf.default到安装目录的etc目录下并重命名 

$ cp ./examples/smb.conf.default  /usr/local/samba/etc/  
$ mv /usr/local/samba/etc/smb.conf.default  /usr/local/samba/etc/smb.conf

2）测试安装成功，切换到/usr/local/samba目录 

$ ./bin/testparm

3）添加用户, 使用以下命令添加一个samba用户，他会提示你输入密码，（samba服务器的用户是依赖于linux的用户，只能添加linux中已存在的用户）

$ ./bin/smbpasswd -a beyond

4）启动samba 服务，安装目录下 

$ sbin/nmbd
$ sbin/smbd

5）测试，在Windows下，执行win+r 输入目标IP地址，输入添加的用户名密码即可共享目标主机文件

0x03 漏洞测试

允许匿名登录测试

在目标机上建一个共享目录 

$ mkdir /home/share   
$ chmod 777 /home/share

编辑Samba配置文件

$ vim /usr/local/samba/etc/smb.conf

在后面增加如下内容，共享目录名为myshare 

[myshare]
comment = test
path = /home/share   #上面创建的共享目录
browseable=yes       #可读
writable = yes         #可写
guest ok = yes       #允许匿名登录

在kali中利用漏洞，漏洞利用完成，从如中可以看到.so文件已经上传到靶机了，但无法建立会话。

查看show targets，我们目标靶机是32位系统，于是我们设置target为1测试

直接报错

再次测试，在Debian8.8.0 64位系统上测试，依然如此

将目标共享目录改成 /home/shared(之前的目录是/home/share), 重启服务 复现成功了…
这里如果我们设置的共享目录在利用代码里面有列出，直接利用就可以，如果我们设置的共享目录在代码里面没有，需要在msf中手动设置SMB_SHARE_BASE

禁止匿名登录测试

更改Samba配置文件，禁止guest匿名登录

我们直接利用会失败，提示没有共享目录，下图为报错信息

我们在msfconsole中配置登录用户名密码即可

利用成功，返回的是登录用户shell
